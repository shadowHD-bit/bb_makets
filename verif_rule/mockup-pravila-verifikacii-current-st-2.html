Правила верификации · полная спецификация (v0.2) →

Это единственное место редактирования правил верификации (витрина + источник истины); на операционных экранах (Верификации, Внешние KYC) они показаны read-only. Любое ручное действие требует причины и пишется в Аудит (событие verification_rule.updated). Правила версионируются; удаление регуляторных правил запрещено (только архив/приостановка); публикация идёт через Релизы (без мгновенной публикации). Статусы Ограничено/Приостановлено деривируются из здоровья провайдера.

Расположение в навигации: уровень платформы

Правила верификации находятся в «Настройки» → «Платформа» (уровень оператора), а не в тенанте. Обоснование: реестр KYC/KYB-уровней регуляторно значим и един для всех тенантов одного контура — тенанты выбирают из утверждённых уровней, но не меняют их самостоятельно. WL-тенанты работают через вкладку SSO-путь / отдельный контур. Перенос в тенант-настройки потребует изменений в router/index.ts и domains.ts.

KYB-1 — базовый юрлица · VR-04

Ограничено

Правило ограничено — деградация провайдера

Провайдер IDX сейчас отвечает с ошибками. Статус деривируется из Интеграций и снимется автоматически после восстановления; ручная проверка остаётся доступна как фолбэк.

Переверификация и SLA

Триггер переверификации

Переверификация стартует по истечении срока (12/24/36 мес.).

Каналы напоминаний (эскалация email → email+TG → SMS)

Дефолты §10: KYC L2 — 4ч→8ч; KYB — 24ч→48ч; «Нужны правки → Истёкшая» — 14 дней; напоминания — 30 дней · 3 попытки.

Требования этого правила (KYB-1 · IDX)

Уровень KYB-1 требует отмеченные проверки; провайдер выполняет их, ручная проверка доступна всегда как канал.

Провайдер по требованию (расширенный режим)

По умолчанию все требования выполняет провайдер уровня (IDX). Для смешанных сценариев (напр. Sumsub для документов + ИПДС для AML) можно назначить отдельного провайдера на каждое требование. Переопределения записываются в сводку изменений и идут в аудит.
